# 主页强制使用压缩图片更新

## 更新说明
根据需求，主页现在**强制使用压缩图片**，在任何情况下都不会使用原图。

## 具体修改

### GalleryGrid.vue 修改

1. **主图片加载**
   ```javascript
   // 修改前：优先使用压缩图，失败时使用原图
   :src="photo.compressedUrl || photo.thumbnailUrl || photo.imageUrl"

   // 修改后：只使用压缩图或缩略图，不使用原图
   :src="photo.compressedUrl || photo.thumbnailUrl"
   ```

2. **悬停预览图片**
   ```javascript
   // 修改前：使用原图
   :src="photo.imageUrl"

   // 修改后：使用压缩图
   :src="photo.compressedUrl || photo.thumbnailUrl"
   ```

## 行为变更

| 场景 | 之前行为 | 现在行为 |
|------|----------|----------|
| 主页图片加载 | 压缩图 → 缩略图 → 原图 | 压缩图 → 缩略图 |
| 悬停预览 | 加载原图 | 加载压缩图 |
| 网络不佳时 | 可能回退到原图 | 绝不使用原图 |

## 影响

1. **性能优化**：主页永远不会加载大体积的原图
2. **用户体验**：即使在网络不佳时，也确保加载较小的图片
3. **数据节省**：减少用户流量消耗

## 注意事项

- 如果压缩图片生成失败，只会使用缩略图
- 缩略图（400px，85%质量）作为最后的备选
- 详情页和模态框仍使用原始高质量图片
